## Frontend Interface

This section describes the frontend interface of the application and how users interact with the database-driven features.

---

## Frontend Goals
The frontend provides a clear interface for users to:
- Manage family members
- Record medical history
- View risk alerts
- Schedule appointments
- Access awareness content

---

## Planned Screens / Pages
- **Home / Dashboard:** Overview of user data and quick access links
- **Family Members:** List, add, edit, and delete family members
- **Medical History:** Add/view medical history records per family member
- **Risk Alerts:** View generated alerts and alert status
- **Clinics & Appointments:** View clinics and schedule/cancel appointments
- **Awareness Content:** Browse educational information

---

## Input Validation & UX
- Required fields must be completed before submission
- User-friendly error messages are shown for invalid input
- Confirmation prompts for delete actions

---

## Notes
UI wireframes were introduced in Phase 1. This phase documents how the UI connects to backend endpoints and database operations.
